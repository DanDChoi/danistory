### 자바의 접근제어자 종류와 차이

#### 1. 접근제어자란?

자바 접근제어자(Access Modifier)는 클래스·필드·메서드·생성자 등 코드 요소가 어디에서 접근 가능한지를 정해주는 키워드이다.
이를 통해 외부에서 내부 데이터를 임의로 바꾸는 것을 막고, 캡슐화와 정보 은닉을 구현해 코드의 안정성과 유지보수성을 높인다.

#### 2. 접근제어자 종류

- public: 어떤 클래스에서라도 접근이 가능
- protected: 동일 패키지내의 클래스 또는 해당 클래스를 상속받은 외부 패키지의 클래스에서 접근이 가능하다.
- default: 접근제어자를 별도로 설정하지 않는다면 default로 설정, 동일 패키지 내에서만 접근이 가능하다.
- private : private이 붙은 변수, 메소드는 해당 클래스에서만 접근이 가능

| 접근제어자 | 외부패키지에서 사용 | 상속 관계에서 사용 | 외부 클래스에서 사용 | 같은 클래스에서 사용 |
| --- | --- | --- | --- | --- |
| public | O | O | O | O |
| protected | X | O | O | O |
| default | X | X | O | O |
| private | X | X | X | O |

> private → default → protected → public 순으로 보다 많은 접근을 허용한다.
>

> 클래스 내의 클래스를 inner 클래스라고 부르는데 이러한 inner 클래스에도 역시 접근제어자를 붙여서 접근을 제어할 수 있다.
>

> 패키지는 ‘폴더’를 말한다.
