### C언어의 구조체와 주소참조
매번 헷갈리던 구조체와 주소참조에 대해 
코드를 통해 한줄씩 분석하며 확실하게 이해해보자.


### 코드 전문
```c
#include <stdio.h>

typedef struct {
		int x;
} Struct;

void change(Struct *a, Struct b) {
		a->x += b.x;
		b.x = 100;
		a = &b;
		a->x = 200;
}

int main() {
		Struct b1 = {3};
		Struct b2 = {4};
		
		change(&b1, b2);
		
		printf("%d", b1.x);
		return 0;
}
```

#### 1. 구조체 정의

```c
typeof struct {
		int x;
} Struct;
```

의미:

x라는 int 하나를 가진 구조체 타입 Struct를 만든다.

즉, 앞으로 Struct는 이러한 형태이다.

```c
Struct
└─ x
```

#### 2. 메인 시작

```c
Struct b1 = {3};
Struct b2 = {4};
```

현재 상태:

```c
b1.x = 3
b2.x = 4
```

#### 3.함수 호출

```c
chang(&b1, b2);
```

함수의 정의는 이렇다.

```c
void change(Struct *a, Struct b)
```

각각 매칭하면

- a 에는 &b1이 들어감
→ a는 b1의 주소
- b 에는 b2가 들어감
→ b는 b2의 복사본

즉, 함수 안에 들어간 직후에는 아래와 같이 생각하면 된다.

```c
a -> b1
b = b2의 복사본
```

값으로 보자면,

```c
b1.x = 3
b2.x = 4
b.x = 4   //복사본
```

#### 4. 첫 줄

```c
a->x += b.x;
```

이 뜻은

> a가 가리키는 구조체의 x에 b.x를 더한다
> 

현재:

- a는 b1을 가리킴
- b.x = 4

따라서:

```c
b1.x = 3 + 4 = 7
```

지금 상태:

```c
b1.x = 7
b2.x = 4
b.x = 4
```

#### 5. 둘째 줄

```c
b.x = 100;
```

헷갈리기 좋은 포인트.

중요한 점:

> b는 b2자체가 아니라 b2의 복사본
> 

따라서 바뀌는 건 복사본 b뿐이다.

변경후:

```c
b1.x = 7
b2.x = 4
b.x = 100
```

즉, b2는 안바뀐다

#### 6. 셋째 줄

```c
a = &b;
```

이게 확실한 함정.

이전까지는:

```c
a -> b1;
```

즉, a가 더이상 b1을 가리키지 않는다.

이제부터 a는 함수 안의 복사본 b를 가리킨다.

현재 상태를 보면:

```c
이전:
a -> b1
이후:
a -> b
```

그리고 값은:

```c
b1.x = 7
b2.x = 4
b.x = 100
```

#### 7.넷째 줄

```c
a->x = 200;
```

지금 a는 누구를 가리키고 있는지 상기한다면

```c
a -> b
```

따라서 이 코드는

```c
b.x = 200
```

과 같다.

즉, 변경 후:

```c
b1.x = 7
b2.x = 4
b.x = 200
```

b1은 더 이상 영향 받지 않는다.

#### 8. 함수 종료

함수가 끝나면 지역변수 a, b는 사라진다.

남는 것:

```c
b1.x = 7
b2.x = 4
```

#### 9.출력

```c
printf("%d", b1.x);
```

출력 값은:

```c
7
```

#### 핵심 압축 정리

함수 들어갈 때

```c
a -> b1의 주소
b -> b2의 복사본
```

실행 흐름

```c
a->x += b.x;  -> b1.x = 7
b.x = 100;    -> 복사본 b만 변경
a = &b;       -> a가 이제 b1말고 b를 가리킴
a->x = 200;   -> b.x = 200
```

최종

```c
b1.x = 7
```

#### 이 문제의 진짜 포인트 2가지

```c
1. Struct b 는 값 전달이라 복사본이다
	2. a = &b; 이후부터는 b1이 아니라 b를 건드린다
```
